# Sprint Retrospectives (reconstructed from Git)

## What worked well
- Strong ability to fix issues.
- Good progress on critical flows: booking, Stripe, dashboard.
- Quick response to admin UI readability issues and business consistency checks (session duration).

## Challenges encountered
- We used Trello at the beginning, but later stopped using it because we communicate directly.
- We communicated in person on Tuesdays and Thursdays, and the rest of the week via WhatsApp or Discord.
- Some fixes had to be chained (domino effect on booking/payment).

## Improvement actions (next sprint)
1. Set up a minimal tracking board (To do / In progress / Review / Done).
2. Link each bug to an ID and a fix commit.
3. Keep a test log (scenario, result, date, owner).
4. Reserve 30 minutes at the end of each day for documentation updates.

## Retro questions (checklist)
- What worked well?
- What blocked us?
- What will we concretely change in the next sprint?
